## Блок 4: API и оффлайн

### Зачёт

Подключить реальный API вместо локального JSON и довести bottom sheet деталей товара до состояния из макета.

**Сборка**
* Проект собирается без ошибок

**Bottom sheet деталей**
* Соответствие макету: отступы, шрифты, цвета, скругления
* Размеры (`sizes`) — рабочий выбор
* Кнопка «В корзину» (пока без логики добавления)
* Кнопка (i) → попап/диалог с дополнительной информацией:
  * Характеристики: материал (`material`), вес (`weight`), сезон (`season`), страна производства (`countryOfOrigin`)

**API**
* Данные каталога загружаются из API, не из локального JSON
* Endpoint: `GET https://fefu2026spring.deploy.feip.dev/catalog`
* Авторизация: заголовок `Authorization: Bearer <token>` (токен — `Cmt7wdwFgDIi1_SRX8hlJIExs0jJKPr4axflLpExAxM`)
* Формат ответа совпадает со структурой [products.json](../hw2/products.json) из второго блока
* Состояние ошибки сети с возможностью повторить загрузку

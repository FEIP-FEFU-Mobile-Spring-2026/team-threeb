Блок 5: Корзина и заказы

Макет: https://www.figma.com/design/aSy8xRchjFo8BY6LlphfDi/Untitled?node-id=0-1&t=VP9vNyR6ig83nhr5-1
Зачёт

Реализовать корзину и оформление заказа. Пользователь добавляет товары из bottom sheet деталей, меняет количество, удаляет позиции и оформляет заказ.

Сборка

    Проект собирается без ошибок

Бейдж

    На иконке корзины в нижней навигации отображается количество товаров в корзине
    Бейдж обновляется при изменении содержимого корзины

Добавление в корзину

    Кнопка «В корзину» в bottom sheet деталей работает
    В корзину добавляется выбранный размер
    Повторное добавление того же товара с тем же размером увеличивает количество (а не создаёт дубликат)

Экран корзины

    Список добавленных товаров
    Элемент корзины содержит:
        Название товара (name)
        Размер (sizes[].name)
        Цена с учётом количества (priceInKopecks × количество) — отформатирована в рублях
        Контрол изменения количества (+/−)
        Кнопка удаления позиции
    Общая стоимость корзины
    Кнопка очистки корзины → модальное окно подтверждения
    Empty state если корзина пустая
    Локальное хранение корзины (SharedPreferences / UserDefaults / аналог)
    В локальном хранилище сохраняются только id товара, id размера и количество
    Содержимое корзины восстанавливается при перезапуске приложения из каталога + локально сохранённых данных

Оформление заказа на экране корзины

    Кнопка «Оформить»
    Кнопка «Оформить» отображается только если в корзине есть товары
    После оформления: очистка корзины + bottom sheet / alert с сообщением об успешном оформлении
